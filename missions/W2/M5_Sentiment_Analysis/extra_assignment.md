# 워드클라우드 작동 원리 (핵심 요약)

`wordcloud` 라이브러리가 빈도 딕셔너리 → PNG를 만드는 로직은 한 문장으로:

> **빈도 내림차순으로 정렬한 단어를, 빈도에 비례한 폰트 크기로, 빈 자리를 찾아 하나씩 박아 넣는 탐욕(greedy) 배치.**

## 4단계 흐름

```
[1] 정렬·정규화 → [2] 폰트 크기 결정 → [3] 빈 자리 탐색·배치 → [4] 색칠·렌더링
```

**[1] 정렬·정규화** — 빈도 내림차순 정렬 후 상위 `max_words`(우리: 100)개만 남기고, 최대 빈도 기준 0~1로 정규화한다. 큰 단어부터 놓아야 배치 실패가 줄기 때문에 순서가 중요하다.

**[2] 폰트 크기 결정** — 완전 빈도 비례가 아니다. 리뷰 데이터처럼 빈도가 치우친 분포에서 비례로 그리면 1위 단어가 화면을 다 먹는다. 그래서 `relative_scaling`(기본 0.5)으로 **빈도 비례 50% + 순위 기반 50%를 절충**한다.

**[3] 배치 (핵심)** — 각 단어마다:

```
방향 결정 (prefer_horizontal=0.9 → 90% 가로)
  → 경계 상자 크기 측정
  → 캔버스에서 "안 겹치는 빈 자리"를 랜덤 탐색
  → 자리 없으면 폰트를 font_step만큼 줄여 재시도
  → min_font_size까지 줄여도 안 되면 그 단어는 조용히 탈락
```

"이 사각형 자리가 비었나?"의 충돌 검사는 **적분 이미지(2차원 누적합)**로 한다. 캔버스를 글자 있음=1/없음=0 행렬로 보고 누적합 테이블 `I`를 만들면, 임의 사각형의 점유량이 `I[D]-I[B]-I[C]+I[A]` 한 번, 즉 **O(1)**에 나온다 (코딩테스트의 2차원 구간합 그대로). 단어를 놓을 때마다 이 테이블을 갱신한다.

**[4] 색칠** — 배치가 끝난 뒤 colormap(긍정=Blues, 부정=Reds)에서 단어마다 **랜덤**으로 색을 뽑는다. 같은 그림 안의 색 농담 차이는 의미가 없다.

## 해석 시 주의 3가지

1. **의미 있는 시각 변수는 글자 크기 하나뿐.** 위치·방향·색·단어 간 근접성은 전부 랜덤이다. ("'배송'이 '불량' 옆에 있으니 연관" = 오독)
2. **그림 간 비교 불가.** 각 그림은 자기 1위 단어 기준 정규화라, 라이언 그림과 춘식이 그림의 글자 크기를 직접 비교하면 안 된다. 정확한 수치는 `keywords_by_character.json`으로.
3. **실행마다 그림이 다르다.** 빈 자리를 랜덤 선택하므로 재현하려면 `WordCloud(random_state=42)` 고정 필요 (현재 미고정 — 개선 백로그).

## 기타

- 파이썬 `wordcloud`는 **랜덤 빈자리 샘플링** 방식. "중앙에서 나선을 그리며 배치"는 원조 Wordle/d3-cloud 쪽 알고리즘이다.
- 한글 폰트 지정(`font_path`)은 미관이 아니라 필수 — 크기 측정과 렌더링 모두 폰트 기준이라 없으면 □(tofu)가 그려진다.
